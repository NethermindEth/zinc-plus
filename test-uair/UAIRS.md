# UAIRs in `zinc-test-uair`

Reference for the UAIRs (Univariate AIR-style constraint systems) defined in
[src/lib.rs](src/lib.rs). Each UAIR specifies a column layout, optional row
shifts, and a set of constraints expressed via the `ConstraintBuilder` DSL.
Constraints take the form `expr âˆˆ ideal` (or `expr == 0` via `assert_zero`).
At verification time, expressions are projected to a prime field and the
ideal-membership check is evaluated there.

## Conventions

- **Column layout** is `(binary_poly, arbitrary_poly, int)` â€” the three column
  kinds supported by `TotalColumnLayout::new`.
- `bp[i]` = binary-poly column `i` (a `BinaryPoly<32>` polynomial â€” 32
  zero/one coefficients).
- `ap[i]` = arbitrary-poly column `i` (a `DensePolynomial<R, 32>`).
- `int[i]` = integer column `i` (a single ring element).
- `up.X` = the column at row `t`. `down.X` = the column at row `t + shift`.
  **Important:** `down.binary_poly` (etc.) is indexed by *position in the
  shifts vector*, not by source column index. With shifts
  `[ShiftSpec::new(0,1), ShiftSpec::new(4,1)]`, `down.binary_poly[0]` is
  `bp[0]` shifted by 1 and `down.binary_poly[1]` is `bp[4]` shifted by 1.
  Shifts are zero-padded beyond the trace boundary.
- `(X - r)` denotes the `DegreeOneIdeal` generated by the polynomial `X - r`.
  An expression is in `(X - r)` iff it evaluates to `0` at `X = r`. For a
  `BinaryPoly`, `eval(2)` is the integer encoded by its bits and `eval(1)` is
  the popcount.
- `ImpossibleIdeal` is a placeholder for UAIRs whose constraints are pure
  `assert_zero` (no ideal projection used).

## `TestUairSimpleMultiplication<R>`

A nonlinear "rotor" UAIR exercising multiplicative constraints across two
adjacent rows.

- **Layout:** 0 binary, 3 arbitrary, 0 int.
- **Shifts:** all 3 arbitrary columns shifted by 1.
- **Ideal:** `ImpossibleIdeal` (constraints are equalities, not ideal checks).
- **Constraints (per row `t`, with `up = ap[*][t]`, `down = ap[*][t+1]`):**
  - `down[0] == up[0] * up[1]`
  - `down[1] == up[1] * up[2]`
  - `down[2] == up[0] * up[2]`
- **Trace:** initial row randomized; subsequent rows generated by the
  multiplicative recurrence above. Asserts `degree < 32` row-by-row to keep
  polynomials within the `DEGREE_PLUS_ONE = 32` bound. Used to test the
  combined-poly prover path on a max-degree-2 UAIR.

## `TestAirNoMultiplication<R>`

The smallest meaningful linear UAIR. *(Marked `// TODO: Rename to XxxUairXxx`
in the source.)*

- **Layout:** 0 binary, 3 arbitrary, 0 int.
- **Shifts:** none.
- **Ideal:** `DegreeOneIdeal<R>`.
- **Constraint (per row `t`):**
  - `ap[0] + ap[1] - ap[2] âˆˆ (X - 2)`
- **Trace:** `ap[0]` and `ap[1]` are random "binary-coefficient" polynomials
  (bits projected into `R`); `ap[2] := ap[0] + ap[1]`. The constraint is
  trivially satisfied because the LHS is identically zero.

## `TestAirScalarMultiplications<R>`

Single-row UAIR demonstrating scalar (`MulByScalar`) and `from_ref`
multiplication of expressions by fixed `DensePolynomial` coefficients.

- **Layout:** 0 binary, 3 arbitrary, 0 int (size derived from `up`).
- **Shifts:** none.
- **Ideal:** `DegreeOneIdeal<R>`.
- **Constraint (per row `t`):**
  - `(X^2 - 1) * ap[0] + ap[1] - ap[2] + (1 + 2X + 3X^2 + 4X^3) âˆˆ (X - 2)`
- **Trace:** no `GenerateRandomTrace` impl. Used as a unit test fixture for
  the prover/verifier paths that exercise `mbs` and `from_ref`.

## `BinaryDecompositionUair<R>`

Connects a `BinaryPoly` column to an integer column representing its value
when bits are read as a base-2 integer.

- **Layout:** 1 binary, 0 arbitrary, 1 int.
- **Shifts:** none.
- **Ideal:** `DegreeOneIdeal<R>`.
- **Constraint (per row `t`):**
  - `bp[0] - int[0] âˆˆ (X - 2)` â€” i.e., `bp[0]` evaluated at `X = 2` equals
    `int[0]`, so the binary column is the bit decomposition of the integer
    column.
- **Trace:** `int[0][t]` is a random `u32`, `bp[0][t] := BinaryPoly::from(int[0][t])`.

## `BigLinearUair<R>`

A "wide and deep" linear UAIR designed for end-to-end benchmarking. Encodes a
popcount accumulator that propagates across rows.

- **Layout:** 16 binary, 0 arbitrary, 1 int.
- **Shifts:** all 16 binary columns shifted by 1 (so `down.binary_poly[i]`
  refers to `bp[i][t+1]`).
- **Ideal:** `DegreeOneIdeal<R>`.
- **Constraints (per row `t`):**
  - `(bp[0] + bp[1] + â€¦ + bp[15]) - int[0] âˆˆ (X - 1)` â€” `int[0]` equals the
    total popcount of the 16 binary columns at row `t`.
  - `down.binary_poly[0] - int[0] âˆˆ (X - 2)` â€” `bp[0][t+1]` is the
    `BinaryPoly` representation of that popcount (so its eval at 2 equals the
    popcount).
  - For `i âˆˆ 1..16`: `up.binary_poly[i] - down.binary_poly[i] âˆˆ (X - 1)` â€”
    columns 1..15 are allowed to permute their bits across rows but must
    preserve their popcount.
- **Trace:** row 0 is fully random; for each subsequent row, `bp[1..16]` get
  fresh random `BinaryPoly`s with the *same* popcount as the previous row,
  and `bp[0]` is set to encode the row's total popcount.

## `BigLinearUairWithPublicInput<R>`

Identical to `BigLinearUair` except it marks the first 4 binary columns as
public input.

- **Layout:** 16 binary, 0 arbitrary, 1 int.
- **Public layout:** `(4, 0, 0)` â€” the first 4 binary columns are public.
- **Shifts / ideal / constraints / trace:** all delegated to `BigLinearUair`.

## `SHAProxy<R>`

A second large linear UAIR, used as an alternative end-to-end benchmark with a
different shape from `BigLinearUair` (more columns, fewer shifts, mixed
binary/int summations, and a multiply-by-`X` constraint).

- **Layout:** 14 binary, 0 arbitrary, 4 int.
- **Shifts:** `bp[0]` shifted by 1 and `bp[4]` shifted by 1 (so
  `down.binary_poly[0] = bp[0][t+1]` and `down.binary_poly[1] = bp[4][t+1]`).
- **Ideal:** `DegreeOneIdeal<R>`.
- **Constraints (per row `t`; `dbp_c1 = bp[0][t+1]`, `dbp_c5 = bp[4][t+1]`):**
  - `dbp_c1 - bp[1] - bp[2] - bp[3] - int[0] - int[1] - int[2] âˆˆ (X - 2)`
  - `dbp_c5 - bp[5] - bp[6] - bp[7] - int[1] - int[2] - int[3] âˆˆ (X - 2)`
  - `bp[8]  - int[0] âˆˆ (X - 2)`
  - `bp[9]  - int[1] âˆˆ (X - 2)`
  - `bp[10] - X * bp[11] âˆˆ (X - 1)` *(`X *` encoded via `mbs(&col, &x_scalar)`
    where `x_scalar = DensePolynomial::new([0, 1])`)*
  - `bp[12] - X * bp[13] âˆˆ (X - 1)`
- **Trace:** `bp[1..=3]` and `bp[5..=7]` are small random binary polys
  (10-bit mask, so each `eval(2) â‰¤ 1023`); `int[0..=3]` are small
  non-negative `u32`s in `0..32`. `bp[8] := BinaryPoly::from(int[0])` and
  `bp[9] := BinaryPoly::from(int[1])` to satisfy the C3/C4 constraints.
  `bp[11]`/`bp[13]` are fully random and `bp[10]`/`bp[12]` are chosen to have
  the same popcount (which makes `bp[k] - X*bp[k+1]` vanish at `X = 1`).
  `bp[0][t+1]` and `bp[4][t+1]` are set to `BinaryPoly::from(S)` where `S` is
  the row-`t` sum that needs to vanish at `X = 2`.

> **Note:** Constraints C5/C6 in the original specification used the ideal
> `(X^32 - 1)` (cyclic ring), but the codebase only ships `DegreeOneIdeal`,
> so this UAIR uses `(X - 1)` instead. This reduces those constraints to a
> popcount-equality between paired binary columns.

## `TestUairMixedShifts<R>`

Minimal UAIR exercising shifts of *different* magnitudes on different columns.

- **Layout:** 0 binary, 3 arbitrary, 0 int.
- **Shifts:** `ap[0]` shifted by 1 (Fibonacci-style), `ap[1]` shifted by 2.
- **Ideal:** `ImpossibleIdeal` (uses only `assert_zero`).
- **Constraints (per row `t`):**
  - `down[0] - up[0] - up[1] == 0` (i.e., `ap[0][t+1] = ap[0][t] + ap[1][t]`,
    a Fibonacci recurrence on column 0)
  - `up[2] - down[1] == 0` (i.e., `ap[2][t] = ap[1][t+2]`, a "look ahead by
    2" relation)
- **Trace:** initial `ap[0][0]`, `ap[1][0..2]` random; the rest derived by
  the recurrence. Used to test multi-shift bookkeeping in the trace
  projection / shift-spec layer.
