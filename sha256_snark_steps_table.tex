\caption{%
SHA-256 compression SNARK prover pipeline, broken into individual steps
(parallel+asm+simd, Apple M1 Max).
The UAIR under test is \texttt{Sha256CompressionUair}, which encodes a single
invocation of the SHA-256 compression function
$h = \mathrm{compress}(H, M)$
over 64 rounds of mixing, with 32-bit additions reduced to binary-polynomial
and integer trace columns.
The PCS layer uses IPRS codes at rate~$1/4$
($\mathbb{F}_{65537}$, R4B32 $D{=}1$ for $n{=}9$; R4B16 $D{=}2$ for $n{=}10$).
%
\textbf{Steps:}
(1)~\emph{PCS commit} -- encode both batches (binary-poly and integer columns)
    into IPRS codewords and Merkle-commit;
(2)~\emph{PCS test} -- produce proximity proofs that committed columns are
    close to valid codewords;
(3)~\emph{Transcript \& field cfg} -- absorb commitment roots and sample a
    random prime-field modulus via Miller--Rabin;
(4)~\emph{Project trace to $F[X]$} -- lift binary-poly and integer trace
    columns to multilinear extensions over~$F[X]$;
(5)~\emph{Project scalars to $F[X]$} -- map UAIR constraint scalars into~$F[X]$;
(6)~\emph{Ideal check} -- run the ideal-check PIOP prover (sumcheck rounds
    verifying that the projected trace satisfies all constraints);
(7)~\emph{Project to $F$} -- evaluate the $F[X]$ trace and scalars at the
    projecting-element challenge, collapsing to concrete field values;
(8)~\emph{Combined resolver} -- combine all evaluated polynomials into a
    single opening claim for the PCS.
}
